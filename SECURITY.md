# Security Policy

## Supported Versions

This project is currently maintained on the default branch. Security fixes are
applied to the latest released code.

| Version | Supported |
| ------- | --------- |
| latest  | ✅ |
| older   | ❌ |

## Reporting a Vulnerability

Please do **not** open public GitHub issues for suspected vulnerabilities.

Instead, report privately via one of the following:

1. GitHub Security Advisory (preferred).
2. Email: `security@salesforce-doc-portal.example`

Include:

- A clear description of the issue,
- Steps to reproduce,
- Potential impact,
- Any proposed mitigation.

We will acknowledge receipt within 3 business days and provide status updates
as triage progresses.

## Disclosure Process

- We investigate and validate the report.
- We develop and test a fix.
- We coordinate disclosure timing with the reporter when possible.
- We publish a fix and release notes.
